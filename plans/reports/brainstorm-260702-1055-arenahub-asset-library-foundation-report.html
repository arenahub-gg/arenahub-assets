■ 05 · Recommendation 05 / 08

Approved design

Ba nguyên tắc.
Một cấu trúc repo.

01

Extract

Trích xuất từ game thật

Round 1 chỉ tạo những gì Snake thực sự dùng, cộng khung để mở rộng. Component API sai sẽ bị game thứ hai ép refactor — đó là cách thư viện trưởng thành.

02

Consumer

Claude là người dùng

Search = Glob/Grep/Read trên tên file mô tả + catalog markdown sinh bằng script. Metadata mức pack, không mức file. Không vector DB, không API.

03

Ship

Game self-contained

Engine và components được vendor (copy vào) lúc generate. Game deploy CDN dạng static, đóng băng phiên bản engine nó dùng. Thư viện tiến hóa tự do.

Repo structure

arenahub-asset/
├── CLAUDE.md          # entry point cho Claude
├── catalog/           # MD sinh tự động
├── assets/
│   └── kenney-{pack}/
│       └── pack.json  # style, tags, license
├── engine/            # game-loop, input, renderer,
│                      # audio-player, asset-loader
├── components/        # score, timer, hud...
├── sdk/
│   ├── arenahub-sdk.ts            # interface
│   └── arenahub-sdk-local-stub.ts # localStorage
├── templates/
│   └── canvas-game/   # Vite + TS starter
├── games/
│   └── snake/         # demo + reference
├── rules/             # round sau
└── docs/

Execution flow — 6 bước

  • Dựng structure + conventions + CLAUDE.md
  • Import 1-2 Kenney pack (sprites + SFX) + script sinh catalog
  • Định nghĩa SDK interface + local stub
  • Build Snake hoàn chỉnh — asset thật, SDK stub, score
  • Trích xuất ngược: generic → engine/ + components/; khung → templates/canvas-game/
  • Acceptance test: session mới, prompt "làm breakout" → game chạy được, dùng thư viện
